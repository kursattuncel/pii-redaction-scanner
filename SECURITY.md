# Security Policy

This project is defensive and educational. It must not include exploit code,
credential theft, stealth, persistence, destructive actions, or live-target
testing features.

## Supported Versions

The `main` branch is the supported development line.

## Reporting a Vulnerability

Please open a private security advisory or email the maintainer with:

- Impact summary
- Reproduction steps using local sample data only
- Suggested remediation, if known

Do not include real credentials, private logs, customer data, or third-party
secrets in reports or examples.
